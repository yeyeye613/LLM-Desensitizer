### 2\. 后续改进路线图

#### 规则引擎优化
  * **优化正则边界**：将手机号正则改为 `\b1[3-9]\d{1,2}\s?\d{4}\s?\d{4}\b`，增加对空格的容忍。
  * **增加前缀/后缀检查**：识别银行卡时，先匹配常见前缀（如 `6225`），识别邮箱时确保匹配到完整的顶级域名（`.com`, `.cn`）。
  * **冲突优先级逻辑**：在 `DesensitizationManager` 中增加**权重筛选**：
    > *逻辑示例：如果一个字符串同时满足 `BANK_CARD` (16-19位) 和 `PASSWORD`，优先判定为 `BANK_CARD`。*

#### 语义能力增强

  * **引入 NLP 分词**：在检测前使用 **HanLP** 或 **Jieba** 进行分词。如果分词结果标注为 `nr` (人名)，则直接标记为敏感。
  * **LLM 异步校验**：
    1.  正则先过一遍（快）。
    2.  对于没匹配到的剩余文本，调用大模型（如 DeepSeek）提问：“以下文本是否包含隐私？请以 JSON 返回”。
    3.  将 LLM 结果补充进 `actualEntities`。

#### 调试代码修正

  * **模糊匹配开关**：在测试类中增加一个选项，允许“只要识别到的文本包含预期文本”即视为通过，用于调试召回率。
